# AroundCities Feed Use Cases (V2)

## Core Philosophy

A Feed represents anything worth sharing around Kuching.

A Feed is not limited to events.

Examples include:

* Events
* Event updates
* Event registrations
* Food discoveries
* Place visits
* Personal experiences
* Business discoveries
* Seasonal greetings
* Featured photos
* Interesting observations

Most feed attributes are optional.

Only the following should be considered core:

* Title
* Description

Everything else depends on the type of content being shared.

---

# 1. Single Day Event

Example:
DIY Sape Competition

Purpose:
Share a one-day activity discovered from an external source.

Data:

* Title
* Description
* Photos (usually promotional poster)
* Place: AEON Mall Kuching Central
* Schedule:

  * 7 June, 3:00 PM
* Source URL
* Source Screenshot (private evidence)
* Source Channel: AEON Mall Kuching Central

# Design Conclusions

A Feed may optionally have:

* Photos
* Places
* Schedule
* Source URL
* Source Evidence
* Source Channel
* Parent Feed

Not every Feed requires all fields.

The system should be designed around flexibility rather than assuming every Feed is an event.
